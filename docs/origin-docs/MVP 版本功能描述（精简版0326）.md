一、产品概述

SmartSleep Alarm 是一款 iOS + watchOS 智能闹铃应用，通过 Apple Watch实时监测用户的睡眠状态，实现 "睡着时闹铃持续响，醒了自动静音，持续确认用户不再睡回去，并支持手势贪睡" 的核心体验。

核心口号：你睡，我响；你醒，我静；你再睡，我再响；你想赖床，快速转动手腕等。

MVP 范围说明：  
本版本仅聚焦智能唤醒核心逻辑与手势贪睡，不区分睡眠/小憩模式，不包含语音播报功能。其他功能将在后续版本逐步添加。

2.2 清醒判断与重响机制

第一阶段：闹铃响起后的清醒确认

- 闹铃响起后，立即开始监测用户的身体动作和心率变化

- 当检测到可能的清醒信号时，进入 3秒确认窗口（固定值）【有待确定】

- 若在3秒内持续检测到清醒信号，则判定用户已醒，立即停止闹铃

- 目标：用户清醒后，闹铃在 5秒内 停止

第二阶段：闹铃停止后的持续监测（防再睡）

- 闹铃静音后，手表继续监测 5分钟

- 若在此期间检测到用户再次入睡（心率下降、体动消失），则自动重新响起闹铃

- 重新响铃后，再次执行第一阶段判断（3秒确认窗口）

- 此过程循环，直到用户保持清醒超过5分钟，或用户手动关闭闹铃

2.3 手势贪睡功能（新增）

核心逻辑：
当闹铃响起时，如果用户希望再睡一会儿，只需用佩戴 Apple Watch
的那只手做出指定手势（如快速转动手腕等），闹铃将立即静音，并在用户自定义的X 分钟后重新响起。
重新响起的闹铃同样支持智能监测和再次贪睡。

  ---------------------------------------------------------------------------------------------------------------------
  功能点         描述                                                                              优先级
  ------------------ ------------------------------------------------------------------------------------- ------------
  手势识别       检测用户用佩戴手表的手做出的指定动作（快速转动手腕等）                       P0

  贪睡触发       检测到有效手势后，立即停止当前闹铃，记录贪睡计时                                     P0

  贪睡计时器     按用户自定义的贪睡间隔（默认5分钟）倒计时，时间到后重新响起闹铃                      P0

  贪睡次数限制   可选功能：允许用户设置最大贪睡次数（如3次），达到后不再响应贪睡手势（后续版本）      P2

  贪睡提示       贪睡触发后，手表以震动或小图标提示用户已进入贪睡模式                                P1
  ---------------------------------------------------------------------------------------------------------------------

手势识别方式建议：

- 快速转动手腕：检测手腕在短时间内快速旋转一定角度

- MVP
  阶段可先实现一种简单可靠的手势（如手腕快速翻转），后续再扩展更多手势选项

贪睡时间设置：

- 用户在创建闹铃时，可自定义贪睡间隔（1-30分钟，默认5分钟）

- 用户也可在全局设置中修改默认贪睡间隔

2.4 默认响铃策略（含贪睡）

  ----------------------------------------------------------------------------------
  场景                                  行为
  ----------------------------------------- ----------------------------------------
  传感器正常，判断用户仍在睡眠          闹铃持续响

  传感器正常，判断用户已醒              闹铃自动静音（延迟 < 5秒）

  闹铃静音后5分钟内，判断用户再次入睡   闹铃自动重新响起

  用户做出指定手势（快速转动手腕等）        闹铃立即静音，X分钟后重新响起

  手势触发后，X分钟内用户再次入睡       闹铃自动重新响起（与智能监测并行）

  传感器数据缺失/不确定                 闹铃继续响（确保不漏叫）

  手表没电/蓝牙断开                     自动降级为普通闹铃

五、MVP 验收标准

1.  ✅ 用户可正常创建、编辑、删除闹铃

2.  ✅ 闹铃响起前30分钟，手表端自动启动后台监测

3.  ✅ 闹铃响起后，监测到清醒信号，5秒内自动静音

4.  ✅ 闹铃静音后，若用户在5分钟内再次入睡，闹铃自动重新响起

5.  ✅ 闹铃响起时，用户快速转动手腕等，闹铃立即静音，并在用户自定义的X分钟后重新响起

6.  ✅ 贪睡期间若用户再次入睡，闹铃仍会按智能监测逻辑重新响起

7.  ✅ 传感器数据缺失时，闹铃正常响铃（不漏叫）

8.  ✅ 用户可在闹铃详情页开启/关闭智能静音，并设置贪睡间隔

9.  ✅ 权限申请流程完整，用户可正常授权
